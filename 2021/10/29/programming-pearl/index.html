Record some typical problems I met in this book, write the solution based on my knowledge and resource on the internet

Q: How to sort 10M US phone numbers with ~1MB memory? If a duplicate number exists, then throw an error.

A1:

  • Use integer to store the phone numbers, since one integer will need 4 bytes to store which means can store 0.25M phone numbers in the memory. First to read number between 0-0.25M to the memory, then 0.25M to 0.5M to the memory…

A2:

  • Use a Bitmap, 1.25MB bit array can show whether a number exists, then just traverse the bitmap

Follow up:

  • If the memory need to be exactly below 1MB, How to implement it?
    A:
  • Two Pass
    Follow up:
    Store all the free number starts with 800, 877, 888 and more, how to sort them in a 1MB space? How to judge if a free number is used before?
  • Use a hashmap, key is the first three digit of the number, value is a bitmap(smaller since it only needs to store the number left)

Q: Give a set with n integer, and a integer t and integer k, how to quickly know whether there is a subset with k integer, and the sum of the integer are less then t?

A: First method is to sort the set array, O(nlogn), The second method is using Quick-Select, O(n)
