lambda calculus vs turing machine

Posted on 2022-04-24

Lambda calculus is a completely different idea compared with turing maching. We grow up in a 1+1=2 world, numbers and symbols construct our whole understanding of the world. But, there is another world made up of functions(only functions.) The variables can be function, the return can be function and number can also be functions. In this article, I would like to introduce it a little bit based on my understanding.

First of all, forget all numbers and symbols, there is no 0, 1, 2… and no + - * /. We do not have them

What we have? Functions, variables, (Applications)parentheses

Then let’s start

  • The basic function

    λx.x

    λ: Function Signifier

    x before dot: variable

    x after dot: function body

    As a whole, Lambda Abstraction

    In Math World: f(x) = x

    In Javascript World: var I = (x) => x

    Now we got I as a function to return the variable it self

  • function call it self

    λf.ff

    In Math World: f() = f(f)

    In Javascript World: var M = f => f(f)

  • True and False

    Again, we do not have true or false in our lambda world, just use true and false to help understanding.

    3 == 3 equals to True, but here is not going to define 3 == 3, is going to define True it self

    In programming language, there are many if language like:

    if (x > 1) {

    p1

    } else {

    p2

    }

    when x > 1, will execute p1, when x <= 1, will execute p2, which means True is to execute p1 and False is to execute p2, there is no concept of True, only to return the function will be execute.

    So for True, In Math world: f(a, b) = a

    On the other hand, f(a, b) = b

    λab.a = True, λ = False

    In Javascript World:

    var K = (a) => (b) => a;

    var KI = (a) => (b) => b;

    While, KI is not needed, we can use K(I)(a)(b), because K(I)(a) will return I, and I(b) = b

  • Changing sequence of variables

    λfab.fba

    This is very intuitive from the equation.

    In Math World: f(a, b) = f(b, a)

    and In JS: C = f => (a) => (b) = f(b)(a)

    This can be connected with False, since swap the variables and this makes return the first after swap equals to return the second before swap.

We defined four basic functions now.

Lambda Calculus Javascript Math Usage
λx.x I = (x) => x f(x) = x return variable
λf.ff M = f => f(f) f() = f(f) self as parameter
λab.a K = (a) => (b) => a f(a, b) = a Execute first(T)
λab.b KI = (a) => (b) => b/K(I)/C(K) f(a, b) = b Execute second(F)
λfab.fba C = f => (a) => (b) = f(b)(a) f(a, b) = f(b, a) Swap variables

Now we are going to do boolean, again there is no boolean concept in lambda, but in order to help understanding .

let T = K, F = KI/K(I)/C(K)

T(1)(2) will be 1, since return the first variable

F(1)(2) will be 1, since return the second variable

As a result, not(T)(1)(2), should return the 2 and not(F)(1)(2) should return the 1

not(T) = F, not(F) = T, T(F)(T) = F, F(F)(T) = T, so not = (p) => p(F)(T) in JS

represented in λp.pFT

Go one step further, not is swapping the code to return, we can let it go back

executing the first one(T) => not executing the first one(not T) == execute the second one(F)

executing the second one(F) => not executing the second one(not F) == execute the first one(T),

if we swap the variables, the first one will be the second one, vice versa.

Look back to the functions, there is a C which will swap the variables, C(T)(a)(b) = F(b)(a), C(F)(a)(b) = T(b)(a)

C can also be the meaning of not

combinator: no free variables

λb.a not a combinator
